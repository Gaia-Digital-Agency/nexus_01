# Technical Audit — russiancaviarhouse.id
**Server:** hostinger · **Audited:** 2026-06-11 · **Status:** redirect:https://www.russiancaviarhouse.id/

## Verified signals
- HTTP/redirects: The site redirects to `https://www.russiancaviarhouse.id/` via a 301 HTTP/2 redirect. Redirects are handled by WordPress. HTTP/3 (QUIC) is supported.
- robots.txt: Present and declares `Sitemap: https://www.russiancaviarhouse.id/sitemap_index.xml`. It disallows `/wp-admin/` (except `admin-ajax.php`), WooCommerce logs, transient files, uploads, and `add-to-cart` URLs. It contains two `User-agent: *` blocks, one with specific disallows and another with an empty `Disallow:`.
- sitemap: `sitemap_index.xml` is declared in `robots.txt` and is accessible. It is generated by Yoast SEO and includes `post-sitemap.xml` and `page-sitemap.xml`. A request for `sitemap.xml` (without `_index`) returns a 404.
- HTTPS/headers: The site serves content over HTTPS. `content-security-policy: upgrade-insecure-requests`, `permissions-policy`, `x-dns-prefetch-control: on`, `etag`, and `vary: Accept-Encoding` headers are present.
- Platform/version: WordPress 7.0, WooCommerce 10.8.1, PHP 8.2.30.
- Plugins/theme/security: Yoast SEO and LiteSpeed Cache plugins are active. JSON-LD schema is present. Security headers `content-security-policy` and `permissions-policy` are present.

## Technical findings
| Area | Finding | Severity |
|---|---|---|
| Robots.txt | Conflicting `User-agent: *` blocks. One block specifies disallows, while a subsequent block for the same user-agent has an empty `Disallow:`, which can lead to inconsistent crawling behavior. | High |
| Caching | Hostinger CDN (`x-hcdn-cache-status: BYPASS`) is bypassing cache for all observed requests, despite LiteSpeed Cache (`x-litespeed-cache: hit`) being active. This indicates a potential misconfiguration where the CDN is not effectively caching content. | High |
| Performance | One of the 301 redirects exhibits a high upstream response time (`x-hcdn-upstream-rt: 2.186`) which can negatively impact initial page load performance. | Med |

# Technical Audit — caviar.id

> **In plain terms (for the team):** Your site is secure (HTTPS) and Google can crawl it effectively (robots.txt, sitemap), with good speed performance thanks to LiteSpeed Cache. The single biggest risk is the exposed WordPress REST API endpoints (`wp-json`), which could potentially be exploited if not properly secured, impacting your site's security.

**Server:** hostinger · **Audited:** 2026-06-11 · **Status:** live

## Verified signals
- HTTP/redirects: Initial request redirects with a 301 to `https://www.caviar.id/`.
- robots.txt: Contains disallows for `wp-content/uploads/wc-logs/`, `woocommerce_transient_files/`, `woocommerce_uploads/`, `add-to-cart` parameters, and `/wp-admin/` (except `admin-ajax.php`). Includes `Sitemap: https://www.caviar.id/sitemap_index.xml`.
- sitemap: `sitemap_index.xml` is accessible (HTTP 200), generated by Yoast SEO, and includes sitemaps for posts, pages, products, product categories, and product tags.
- HTTPS/headers: Site serves over HTTPS. `content-security-policy: upgrade-insecure-requests` header is present. HTTP/2 is used, and HTTP/3 (alt-svc: h3) is supported. LiteSpeed Cache is active and serving cached content.
- Platform/version: WordPress 7.0, WooCommerce 10.8.1, PHP 8.2.30. Hosted on Hostinger with hPanel.
- Plugins/theme/security: Yoast SEO and LiteSpeed Cache are active. JSON-LD schema is present. WordPress REST API endpoints (`wp-json`) are exposed.

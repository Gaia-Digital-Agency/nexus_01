# Technical Audit — gaiada.com
**Server:** hostinger · **Audited:** 2026-06-11 · **Status:** live

## Verified signals
- HTTP/redirects: HTTP to HTTPS redirect is in place (e.g., `http://gaiada.com` redirects to `https://gaiada.com/`).
- robots.txt: Contains `User-agent: *`, `Crawl-Delay: 20`, and `Sitemap: https://gaiada.com/sitemap.xml`.
- sitemap: `sitemap.xml` is an accessible (HTTP 200) sitemap index generated by Yoast SEO, listing `post-sitemap.xml`, `page-sitemap.xml`, `portfolio-sitemap.xml`, `service-sitemap.xml`, and `category-sitemap.xml`.
- HTTPS/headers: Site is served over HTTP/2 and advertises HTTP/3 (QUIC) support. Headers include `content-type: text/html; charset=UTF-8`, `vary: Accept-Encoding`, `content-security-policy: upgrade-insecure-requests`. Hostinger CDN (`hcdn`) is active.
- Platform/version: WordPress 7.0, PHP 8.1.34. WordPress REST API endpoints (`/wp-json/`) are exposed.
- Plugins/theme/security: Polylang plugin is indicated by a cookie (`pll_language`). JSON-LD schema markup is present. Cookies are set with `secure` and `SameSite=Lax` attributes. A `Content-Security-Policy` header with `upgrade-insecure-requests` is present.

## Technical findings
| Area | Finding | Severity |
|---|---|---|
| robots.txt | The `Crawl-Delay: 20` directive is excessively high, which can severely restrict search engine crawling frequency and impact indexation. | High |
| Speed/Caching | The Hostinger CDN reports `x-hcdn-cache-status: DYNAMIC` for the homepage, suggesting the page is not being cached at the CDN edge, potentially leading to slower load times for repeat visits and increased origin server load. | Medium |
| Platform/Version | While WordPress 7.0 is current, PHP 8.1.34 is not the latest stable version (e.g., PHP 8.2 or 8.3 are available). Keeping PHP updated is important for performance, security, and compatibility. | Low |

## Could not verify
- CMS/platform plugin/theme stack conflicts or bloat (beyond presence of Yoast SEO and Polylang).
- Specific plugin/theme update status.
- Security vulnerabilities related to table prefix, debug mode status, or exposed sensitive files (beyond standard WordPress REST API endpoints).
- Authentication mechanisms.
- Detailed speed/Core Web Vitals metrics (beyond HTTP/2, HTTP/3, CDN presence, and dynamic cache status).
- Full indexability status (e.g., via Google Search Console).
- Google Search Console verification status.

## Top technical fixes (analysis only — NOT executed)
1. Review and remove or significantly reduce the `Crawl-Delay: 20` directive in `robots.txt` to allow for more efficient crawling by search engines.
2. Investigate Hostinger CDN configuration for the homepage to ensure optimal caching, moving from `DYNAMIC` to a more aggressive caching strategy where appropriate to improve load times.
3. Monitor WordPress and PHP versions for new updates and plan upgrades to maintain security and performance.

# Technical Audit — cascadessuites.com

> **In plain terms (for the team):** Your site is generally secure and Google can crawl it well, but the missing main heading (H1 tag) on your homepage is a significant risk that could hurt your search rankings for key terms. While your site uses a CDN for speed, the origin server response times are sometimes slow, which could impact user experience and Google's perception of site speed. The single biggest risk is the **missing H1 tag on the homepage**, which directly affects how well Google understands and ranks your most important page.

**Server:** hostinger · **Audited:** 2026-06-11 · **Status:** live

## Verified signals
- HTTP/redirects: Initial request (implied non-www HTTP) redirects via 301 to `https://www.cascadessuites.com/`. The redirect is handled by WordPress.
- robots.txt: Present and allows all user-agents (`Disallow:`). Declares `Sitemap: https://www.cascadessuites.com/sitemap_index.xml`.
- sitemap: `sitemap_index.xml` is present and valid, listing `page-sitemap.xml` and `room-sitemap.xml`. Generated by Yoast SEO.
- HTTPS/headers: Site serves over HTTPS (HTTP/2). `content-security-policy: upgrade-insecure-requests` header is present. `x-dns-prefetch-control: on` is present. `alt-svc: h3=":443"` indicates HTTP/3 support.
- Platform/version: WordPress 6.7.5. PHP/8.2.30.
- Plugins/theme/security: Yoast SEO plugin is active (indicated by robots.txt and sitemap). LiteSpeed Cache plugin is active (indicated by `x-litespeed-tag`). WordPress handles redirects. `content-security-policy: upgrade-insecure-requests` is present.
- schema presence: JSON-LD schema is present.
- speed/CWV signals: Hostinger CDN (hcdn) is active and caching requests (`x-hcdn-cache-status: HIT/DYNAMIC`). Origin response times (`x-hcdn-upstream-rt`) are observed between 0.630s and 1.738s.
- indexability + GSC-verification STATUS: `robots.txt` allows all, `sitemap_index.xml` is declared.

## Technical findings
| Area | Finding | Severity |
|---|---|---|
| On-Page Structure | Homepage is missing an `<h1>` tag. | High |
| Sitemap | `sitemap.xml` at the root returns an HTTP 404, while `sitemap_index.xml` is correctly declared in `robots.txt` and functions. | Low |
| Platform | WordPress REST API endpoints (`wp-json`) are exposed in HTTP headers. | Info |
| Server/Hosting | Site is hosted on Hostinger with hPanel and utilizes Hostinger CDN (hcdn). | Info |
| Server/Hosting | PHP version 8.2.30 is in use. | Info |
| Server/Hosting | HTTP/3 (h3) is supported. | Info |

## Could not verify
- Plugin/theme stack: conflicts/bloat/updates (beyond presence of Yoast SEO and LiteSpeed Cache)
- Security: table prefix/debug/exposed files/authentication mechanisms (beyond CSP and exposed REST API)
- GSC-verification STATUS (only presence of sitemap/robots.txt verified)
- Comprehensive speed/CWV metrics (only origin response times and CDN usage verified)

## Top technical fixes (analysis only — NOT executed)
1.  Add an `<h1>` tag to the homepage for improved SEO and accessibility.
2.  Address the `sitemap.xml` 404 error by either redirecting it to `sitemap_index.xml` or ensuring it is not referenced by any internal or external links.

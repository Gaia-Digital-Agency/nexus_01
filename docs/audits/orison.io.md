# Technical Audit — orison.io
**Server:** hostinger · **Audited:** 2026-06-11 · **Status:** live

## Verified signals
- HTTP/redirects: The site implements a 301 redirect from initial requests to the canonical `https://www.orison.io/` URL. This redirect is handled by WordPress. The final destination returns an HTTP/2 200 OK status.
- robots.txt: The `robots.txt` file is present, disallows `/wp-admin/` for all user-agents, but explicitly allows `/wp-admin/admin-ajax.php`.
- sitemap: `robots.txt` declares two sitemaps: `https://www.orison.io/sitemap.xml` and `https://www.orison.io/sitemap.rss`. The `sitemap.xml` is accessible (HTTP 200) and is an index generated by All in One SEO v4.9.7.2, containing `post-sitemap.xml` and `page-sitemap.xml` with recent `lastmod` dates.
- HTTPS/headers: The site uses HTTPS and supports HTTP/2 and HTTP/3 (alt-svc). Key headers include `x-powered-by: PHP/8.2.30`, `content-security-policy: upgrade-insecure-requests`, `permissions-policy`, `vary: Accept-Encoding`, and `etag`. Caching is active and hitting (`x-litespeed-cache: hit`, `x-hcdn-cache-status: HIT`).
- Platform/version: The site runs on WordPress (inferred from `x-redirect-by`, `wp-json` links, and AIOSEO plugin) with PHP version 8.2.30. It is hosted on Hostinger (`platform: hostinger`, `panel: hpanel`) and uses Hostinger's CDN (`server: hcdn`).
- Plugins/theme/security: All in One SEO (AIOSEO) v4.9.7.2 and LiteSpeed Cache are active. WordPress REST API endpoints (`/wp-json/`) are exposed. `robots.txt` restricts `/wp-admin/`. Security headers `content-security-policy` and `permissions-policy` are present. JSON-LD schema is present.

## Top technical fixes (analysis only — NOT executed)
1.  **Security: Hide PHP version:** Configure the web server or PHP to suppress the `x-powered-by` header to prevent exposing the PHP version, reducing information available to potential attackers.
2.  **Security: Limit WordPress REST API exposure:** Evaluate if the WordPress REST API endpoints (`/wp-json/`) need to be fully public. If not, consider limiting access or disabling unused endpoints to reduce the potential attack surface.
3.  **Sitemap: Review `sitemap.rss` declaration:** Remove the `Sitemap: https://www.orison.io/sitemap.rss` directive from `robots.txt` if it's not intended to be a valid sitemap index for search engines, as it's a non-standard format for this purpose. Ensure only the primary `sitemap.xml` is declared.
